REBL - Revenue Expense Balance Ledger



Summary

REBL is a financial dashboard web application, designed to generate dynamic reports for a wide range of users and groups. The application will be implemented both in and outside of the Institute of Neuroimaging and Informatics.


The problem

Principal investigators, project managers, and fund managers require a unified financial system that can generate customized reports. Traditional spreadsheet documents can be heavy, static, and difficult to use.

Working with a project manager and a developer, I was tasked with designing the dashboard interface and data visualization.


User research

I met with several stakeholders and was able to identify requirements unique to each user type. Examples:

Principal investigators

  • View budget, cost, and balance for
    • Complete portfolio
    • Individual accounts
  • Filter data by time
    • year-to-date
    • all years
    • fiscal year
    • custom date range
  • Review and allocate accounts for personnel payroll
  • View and edit user profile and settings

Administrators

  • Enter data in back-end
  • Receive alerts for errors
  • Budget planning
  • Expense reconcile
  • Generate detailed reports for auditing
  • Create project, account, user

Wireframes

Not only different essential components had to be present, they also had to be in a clean and functional layout. I started by blocking out labeled areas to represent each important component (left screen). This low-fidelity wireframe allowed me to move things around rapidly for pure layout purpose. I then added pertinent content to reach a more detailed wireframe (right screen) for dicussion with stakeholders.


Preliminary meetings led to additional iterations for different usage patterns, such as choosing time or viewing personnel data. When I created the personnel wireframe, I realized I could (and should) utilize the same layout I used for the portfolio view, specifically for the timeline and account list. This made for a smoother experience switching between different views (portfolio vs. personnel).

Color mockups and iterations

During this stage, several iterative changes occured as I met with users. New content was provided (and required), so the design changed adaptively. For example, the personnel dashboard evolved to be more complex over time, as users essentially asked for it to look like "an Excel table". To keep this simple, I faded out repeated elements ($ sign, same number each row, 0 values).


Version 4 above got closer to what we needed. However, it looked busy. A closer analysis showed that there were 7 distinct blocks (in red, bottom), and there was not enough contrast overall. This led to version 5, where the vertical blocks were reduced to just 3, and contrast was subtly increased:
